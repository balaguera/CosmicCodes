This function finds the index of the neighbour cells for all cells in a mesh.

Parameters
NftNumber of cells/dimension
cells_indexID of cells, container of size \(N_{ft}^{3}\)
excludeif true, the current cell is ignored

This function aims at being used inside a loop within the calculation of mach number and local overdensityies (see <Catalog> Class::Catalog.get_local_mach_number_chuncks()) in order to avoid high RAM consumption, which is the case of using laptops instead of a powerful machine or cluster. Interpolation on a mesh using NGP mass assignment scheme.

weightProperty to use as weight

This method uses a transformation of the coordinates described as follows.

Note
Transformation of coordinates to embed the sample in a cubical box of size \(V=L^{3}\) is \( X_i - > X_i - X_{Offset} + L/2 \). The second term ( \(X_{Offset}\)) is \( X_{Offset}= (X_{max} + X_{min})/2\) places the mid point as the origin of coordinates. The third one places the center of the sample in the center of the cube For a simulation with the point origin (0,0,0) at the lower corner of the box, \(X_{Offset}=Y_{Offset}=Z_{Offset}=L/2\) no real translation is done. If the box has its center at the origin of coordinates, Xoffset=0 and then transformation moves the center of the box to \((L/2,L/2,L/2)\) If Lbox is chosen as xmax-xmin (since it is the largest side) then \( X_{Offset}- L/2 = xmin\) and we end with \(x=x-x_{min}\) but this is not the case for the other components, e,g. \( Y_{Offset} -L/2 = y_{min} -(Lx-Ly)/2 \) Therefore, if the sides of the box are not equal (but still we aim at embedding the survey inside a cube), we need to transform with \(x - > x - (X_{Offset}-L/2)\) instead of subtracting the minimum. This embeds the sample in a box in which, if Lbox is from x, \(x_{min} =0\) \(y_{min}=0\), \(z_{min}=0\) but the actual boundary of the survey, say, in the y direction, is not \fy_{min}\f, but at \(y=(Lx-Ly)/2\). Same applies for CIC, TSC and PCS interpolation schemes.
